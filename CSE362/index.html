General Information

The course covers a range of topics integral to the design, implementation, and testing of a medium-scale software system with the practical experience of implementing such a project as a member of a programmer team. The course also includes material on the social context of computing as well as professionalism and ethical responsibilities in software development.

At the end of this course, the student should understand the complexity of software development projects and the ongoing lifecycle. In particular, they will be able to apply multiple design and implementation methodologies with a sound understanding of quality control and assurance through testing and defect resolution. They will have been exposed to several real world use cases exposing the challenges and impacts of decisions in both the development process and software lifecycle; including the consideration of social and ethical impacts.

Academic Integrity

Academic misconduct, including plagiarism, cheating, and other violations of academic ethics, is a serious academic infraction for which penalties may range from a recorded caution to expulsion from the College. RMC Academic Regulations Section 23 defines plagiarism as: "Using the work of others and attempting to present it as original thought, prose or work. This includes failure to appropriately acknowledge a source, misrepresentation of cited work, and misuse of quotation marks or attribution." It also includes that "the failure to acknowledge that work has been submitted for credit elsewhere." All students should consult the published statements on Academic Misconduct contained in the Royal Military College Undergraduate Academic Regulations.

1. Project Overview

The primary goal of this series of labs is to work as a team in the complete process of developing a game application using Python. This project aims to provide hands-on experience in both the creative and technical aspects of software development. By the end of the project, you will have created a fully functional game, building proficiency in programming, design, and project management.

Skills to be Acquired and Demonstrated

  • Python Programming: Developing Python scripts, applying object-oriented programming principles if needed.
  • Game Design and Logic: Creating engaging game mechanics, narrative design, and implementation.
  • Graphical User Interface (GUI) Development: Utilizing Python GUI libraries for creating interactive and visually appealing interfaces.
  • Database Integration: Using MySQL for data persistence related to game elements like scores and player progress.
  • Version Control with Git/GitHub: Managing code versions and collaborating using Git and GitHub.
  • Testing and Debugging: Ensuring game functionality and reliability through systematic testing and debugging.
  • Documentation and Reporting: Documenting the development process and preparing reports or presentations.

Project Phases

The project will progress through distinct phases including design, development, testing, and presentation. Each phase focuses on different aspects of game development, from conceptualization to final implementation.

Expected Outcomes

By the end of this project, you are expected to have a complete, functional game and a solid understanding of the technologies and techniques used in its development.

2. Choosing your game

Factors to Consider

  • Basic Game Functionality: Aim for clear and achievable game objectives.
  • User Interface: Plan for a user-friendly and engaging interface.
  • Data Persistence: Consider a game that requires saving and loading data.
  • Project Scope: Ensure the game can be completed within our semester timeline.

Game Ideas - you can come up with your own

Classic Arcade Game Clone (e.g., Tetris, Snake, Pong)
Recreate a beloved arcade game. Focus on game loops, collision detection, and user input.
Quiz or Trivia Game
A question-answer based game with various categories and difficulty levels.
Platformer or Adventure Game
A 2D game focused on navigating levels, collecting items, and avoiding obstacles.
Puzzle Game
Develop a game that challenges players with engaging puzzles or logic problems.
Educational Game
Create a game that combines fun with learning, like language or math puzzles.
Role-Playing Game (RPG) - Simplified
A basic RPG with character development, quests, and inventory management.
Strategy Game
Design a game that involves strategic thinking, like resource management or turn-based tactics.

Remember, the goal is to learn and have fun! Choose a project that you will enjoy building and that matches your learning objectives.

3. Understanding the Technology Stack

For our game development project, we will be using a specific set of technologies. This section will briefly explains each component of the technology stack.

Programming Language

Python: A programming language you are familiar with. It is great for quick development and has many libraries that make game development easier.

Graphical User Interface (GUI)

Tkinter/Pygame: These are libraries in Python used for creating the game's visual elements. Tkinter is great for simple 2D graphics, while Pygame provides more advanced graphics and game controls.

You may use other GUI libraries if you wish, or alternatively work with the trio html/css/Javascript to build a web-based interface instead.

Database

MySQL: A database system for storing game data like high scores, player progress, or questions. It is used to save and retrieve data as the game runs.

Version Control

Git/GitHub: Git is a tool for tracking changes in your code, and GitHub is an online platform to store your Git projects. They are essential for collaboration and managing different versions of your game project.

This technology stack is chosen to provide a foundation for your game development while keeping the learning curve manageable.

4. Marking Scheme

Evaluation Elements

  1. Requirements Analysis Document (RAD) - 10%: Quality and completeness of the initial project requirements.
  2. Software Design Document (SDD) - 10%: Detail and clarity in the game design documentation.
  3. Prototype Demo - 15% : Proper intgration of GUI and database with basic interface and minimal functionality.
  4. GitHub Use and Activity - 25%: Effective use of GitHub for version control, including regular commits and contributions.
  5. Final Demo - 25%: Overall functionality and quality of the final game demo.
  6. Final Presentation - 15%: Effectiveness of the final project presentation, including clarity, organization, and demonstration of learning outcomes.

Peer Review and Individual Contribution Assessment

A peer review will be conducted to assess each team member's contribution. Additionally, individual GitHub activity will be reviewed to adjust marks based on each member's involvement and contributions to the project.
