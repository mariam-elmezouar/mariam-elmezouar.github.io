Data Challenge 2

CSE451 Fall 2023

Task: Predict Mobile Price Ranges

Problem: You are launching a mobile company and need to competitively price your phones against giants like Apple and Samsung. Using sales data from various companies, the aim is to link mobile features, such as RAM, to price ranges using a machine learning model. This will help you set your mobile prices more strategically in the market.

Objective: provide a notebook report that 1) explore and visualize the dataset (document any insights gained from the exploration), and 2) build a prediction model to predict the price range of mobile phones based on the features provided.

  • Launch Jupyter Notebook and create a new textbook, and name it DC2_lastname.
  • Download the provided dataset. The dataset contains a list of mobiles phones, with associated features and price range. The columns of the dataset are described below:
    • id - ID
    • battery_power - Total energy a battery can store in one time measured in mAh
    • blue - Has bluetooth or not
    • clock_speed - Speed at which microprocessor executes instructions
    • dual_sim - Has dual sim support or not
    • fc - Front Camera mega pixels
    • four_g - Has 4G or not
    • int_memory - Internal Memory in Gigabytes
    • m_dep - Mobile Depth in cm
    • mobile_wt - Weight of mobile phone
    • n_cores - Number of cores of processor
    • pc - Primary Camera mega pixels
    • px_height - Pixel Resolution Height
    • px_width - Pixel Resolution Width
    • ram - Random Access Memory in Megabytes
    • sc_h - Screen Height of mobile in cm
    • sc_w - Screen Width of mobile in cm
    • talk_time - Longest time that a single battery charge will last when you are talking
    • three_g - Has 3G or not
    • touch_screen - Has touch screen or not
    • wifi - Has wifi or not
    Target Variable:
    • price_range - This is the target variable with value of:
      • 0 - low cost
      • 1 - medium cost
      • 2 - high cost
      • 3 - very high cost

    Your tasks:

  • Check for duplicates/missing values/inconsistant data types. If you comes across any of these issues, fix it and explain what your fix is.
  • Answer the following questions based on the dataset:

    Univariate Analysis (answer 2 out of 4):

    • Battery Power:
      • What is the distribution of battery power across all mobiles?
      • Identify the mobiles with the top 5% of battery power.
    • RAM:
      • How is the RAM distributed? Is it skewed towards lower or higher values?
      • What is the average RAM size across the dataset?
    • Dual SIM:
      • What proportion of mobiles supports dual SIM?
      • Is there a particular price range where dual SIM is more common?
    • 4G Support:
      • What percentage of mobiles have 4G support?
      • Which price range has the least number of 4G supported mobiles?

    Bivariate Analysis (answer 2 out of 4):

    • RAM vs. Battery Power:
      • Is there any observable relationship between RAM size and battery power?
      • Which phones (top 5) have both high RAM and high battery power?
    • 4G Support vs. Dual SIM:
      • Is there a pattern where phones with 4G support more commonly have dual SIM features or vice versa?
    • Battery Power vs. Price Range:
      • How does the battery power distribution vary across different price ranges?
      • Is there a noticeable difference in average battery power between the lowest and highest price range?
    • RAM vs. Price Range:
      • How does RAM distribution vary across price ranges?
      • Is there a clear distinction in RAM size when moving from a lower to a higher price range?

    Prediction of Price Range:

    Implement a Logistic Regression model or Random Forest model to predict price ranges.
    Split the dataset into training and testing sets.
    Train the model on the training data.
    Evaluate the model's performance using appropriate classification metrics (e.g., accuracy, F1-score, precision, recall) on the testing dataset.
    Interpret the results and assess the model's strengths and weaknesses.

  • Once happy with your analysis, clean up the notebook and add explanation/comments/insights as appropriate. Then, submit your notebook on Moodle.
