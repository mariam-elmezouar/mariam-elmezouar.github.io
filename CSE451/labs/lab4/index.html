Introduction

In this lab, the objective is to consolidate what you have done so far, and to build a plan for the modeling stage of the process.

Objectives

  • Write up your steps and results in exploring/cleaning/preparing the dataset.
  • Perform a simple modelling task on the data.
  • Come up with a plan to tackle the selected research questions.

Report

It's time to document the work you've completed so far. This will form the initial part of the report you'll be turning in come December. You can generate this document using a Jupyter notebook. With some cleanup, it can be transformed to resemble a formal report (see examples here and here). Alternatively, you can use your regular text editor for this purpose and submit a PDF file. The report should contain the following sections:

  1. Introduction:

    Provide a brief introduction to the report. Explain the purpose of the report and what the reader can expect to find in it.

  2. Dataset Description:

    Begin by describing the dataset you were given. Include the following information:

    • Dataset name and source.
    • The number of records (rows) and features (columns) in the dataset.
    • A brief overview of what the dataset represents or its context.

  3. Data Cleaning:

    Detail the data cleaning steps you performed in previous labs. Include the following:

    • Identification and handling of missing values.
    • Handling of duplicates (if any).
    • Correcting data type inconsistencies.
    • Addressing outliers (if applicable).
    • Any other data cleaning activities specific to your dataset.

  4. Data Exploration:

    Discuss your data exploration efforts. Include the following:

    • Summary statistics of key variables (mean, median, standard deviation, etc.).
    • Good quality and easily readable visualizations (e.g., histograms, scatter plots, box plots) used to explore data distributions and relationships.
    • Any patterns, trends, or anomalies you observed during exploration.

  5. Data Preparation:

    Describe how you prepared the data for analysis. Include the following:

    • Feature engineering: Any new features created or transformations applied.
    • Encoding categorical variables (if necessary).
    • Scaling or standardizing numerical features (if applicable).
    • Handling imbalanced datasets (if relevant).

  6. Challenges and Solutions:

    Reflect on any challenges or issues you encountered during data exploration, cleaning, or preparation. Explain how you addressed these challenges.

  7. Insights and Discoveries:

    Highlight any interesting insights or discoveries you made during the process. What did you learn about the dataset, and how might these insights inform your future analysis?

  8. Data Quality Assessment:

    Provide an assessment of the overall data quality after the cleaning and preparation steps. Mention any remaining data quality concerns.

  9. Conclusion:

    Summarize the key takeaways from your data exploration, cleaning, and preparation efforts. Emphasize the importance of these steps in the data science process.

  10. Future Steps:

    Mention what you plan to do next, which includes data modeling and evaluation. How will the work you've done so far inform your modeling choices and research questions?

  11. References (if applicable):

    If you referred to any external sources, data documentation, or literature during your work, include them in a references section.

  12. Appendices:

    Include any additional tables, charts, or code snippets that support your findings and processes.

Preliminary modeling task: Predicting Attack Types using Logistic Regression and Random Forest

  1. Objective:

    In this task, you will build and compare two different models, Logistic Regression and Random Forest, for the multi-class classification problem of predicting attack types based on the provided Global Terrorism Database.

  2. Steps:
    1. Data Preprocessing:

      Load and preprocess the Global Terrorism Database, focusing on relevant columns, including the "attacktype1_txt" column (target) and other relevant features.

    2. Data Exploration:

      Explore the dataset to understand the distribution of attack types and visualize relationships between features and the target variable.

    3. Model 1: Logistic Regression

      Implement a Logistic Regression model to predict attack types.
      Split the dataset into training and testing sets.
      Train the Logistic Regression model on the training data.
      Evaluate the model's performance using appropriate classification metrics (e.g., accuracy, F1-score, precision, recall) on the testing dataset.
      Interpret the results and assess the model's strengths and weaknesses.

    4. Model 2: Random Forest

      Conduct independent research to understand the Random Forest algorithm and its application to multi-class classification.
      Implement a Random Forest model for predicting attack types.
      Split the dataset into training and testing sets.
      Train the Random Forest model on the training data.
      Experiment with different hyperparameters (e.g., number of trees, tree depth) and feature selection strategies to optimize model performance.
      Evaluate the Random Forest model's performance using appropriate classification metrics on the testing dataset.
      Document your experimentation process and the chosen hyperparameters.

    5. Model Comparison:

      Compare the performance of Logistic Regression and Random Forest models in terms of accuracy, F1-score, precision, recall, and any other relevant metrics.
      Provide insights into which model performed better and discuss possible reasons for the observed differences.

    6. Conclusion and Reflection:

      Summarize your findings and insights from using both models.
      Reflect on the strengths and weaknesses of Logistic Regression and Random Forest for this specific multi-class classification task.
      Discuss any challenges encountered during the model implementation and evaluation.
