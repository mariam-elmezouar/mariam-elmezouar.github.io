Task 1:

  • Client (noClient, name, noTel, address)

Exercise 2:

Consider a relational database containing information about employees. It has several tables:

  • Employee (NSS, Name, First name, Date of Birth, Address, Salary, NDep, Superior)
  • Department (NDep, NomD, Director)
  • Project (NPro, NomP, Location, NDep)
  • Works (NSS, NPro, Hours)

The foreign key constraints are as follows:

  • Employee (Superior) references Employee (NSS)
  • Employee (NDep) references Department (NDep)
  • Department (Director) references Employee (NSS)
  • Project (NDep) references Department (NDep)
  • Works (NPro) references Project (NPro)
  • Works (NSS) references Employee (NSS)

The Superior attribute in the Employee relationship contains the social security number of the employee’s direct supervisor. Each employee belongs to a department and works on one or more projects. Each project is attached to a department which may be different from that of the employees working on this project.
