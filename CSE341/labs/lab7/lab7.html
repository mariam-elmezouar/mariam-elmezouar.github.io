Case study:

A library management system database is being designed to help the library director and staff efficiently manage and track the vast collection of books, authors, categories, and borrowers. The system facilitates the management of book borrowing transactions. This system aims to enhance the overall library management process, from maintaining an organized catalog of books to managing borrower transactions and ensuring the timely return of borrowed materials.

Database Schema: Library Management System

1. authors Table
  • author_id INT AUTO_INCREMENT (PK): Unique identifier for the author.
  • name VARCHAR(255): The name of the author.
  • bio TEXT: A short biography of the author.
2. categories Table
  • category_id INT AUTO_INCREMENT (PK): Unique identifier for the category.
  • name VARCHAR(100): The name of the category (e.g., Fantasy, Dystopian).
  • description TEXT: A description of what the category encompasses.
3. books Table
  • book_id INT AUTO_INCREMENT (PK): Unique identifier for the book.
  • title VARCHAR(255): The title of the book.
  • author_id INT (FK): Reference to the authors table.
  • category_id INT (FK): Reference to the categories table.
  • isbn VARCHAR(13): The International Standard Book Number.
  • published_date DATE: The date when the book was published.
  • status ENUM(‘available’, ‘borrowed’): The current status of the book.
4. borrowers Table
  • borrower_id INT AUTO_INCREMENT (PK): Unique identifier for the borrower.
  • name VARCHAR(255): The name of the borrower.
  • email VARCHAR(255): The email address of the borrower.
5. book_borrowers Table
  • book_id INT (FK): Reference to the books table.
  • borrower_id INT (FK): Reference to the borrowers table.
  • borrowed_date DATE: The date when the book was borrowed.
  • due_date DATE: The date by which the book is due to be returned.
  • returned_date DATE: The date when the book was actually returned.
  • Primary Key (PK): Composite key consisting of book_id and borrower_id.

import mysql.connector
from tkinter import Tk, Label, Entry, Button, messagebox

# Database connection class
class LibraryDB:
    def __init__(self, host, user, password, database):
        self.connection = mysql.connector.connect(
            host=host,
            user=user,
            password=password,
            database=database
        )
        self.cursor = self.connection.cursor()
    
    def add_borrower(self, name, email):
        try:
            sql = "INSERT INTO borrowers (name, email) VALUES (%s, %s)"
            self.cursor.execute(sql, (name, email))
            self.connection.commit()
            return True
        except mysql.connector.Error as err:
            print("Error: ", err)
            return False

# GUI class
class LibraryApp:
    def __init__(self, root):
        self.db = LibraryDB('localhost', 'username', 'password', 'library_db')
        self.root = root
        self.root.title("Library Management System")

        # Add GUI components here...
        self.name_label = Label(root, text="Name:")
        self.name_label.pack()

        self.name_entry = Entry(root)
        self.name_entry.pack()

        self.email_label = Label(root, text="Email:")
        self.email_label.pack()

        self.email_entry = Entry(root)
        self.email_entry.pack()

        self.add_button = Button(root, text="Add Borrower", command=self.add_borrower)
        self.add_button.pack()

    def add_borrower(self):
        name = self.name_entry.get()
        email = self.email_entry.get()
        if self.db.add_borrower(name, email):
            messagebox.showinfo("Success", "Borrower added successfully!")
        else:
            messagebox.showerror("Error", "Failed to add borrower.")

if __name__ == "__main__":
    root = Tk()
    app = LibraryApp(root)
    root.mainloop()
