Levels of Programming Languages

Programming languages are often categorized into two main levels: high-level languages and low-level languages.

High-Level Languages

High-level languages are programming languages that are closer to human languages and further from machine languages. They are designed to be easy to read and write, making them more accessible to programmers. Examples include Python, Java, C#, and JavaScript.

Advantages

  • Ease of Use: More straightforward syntax, resembling natural language.
  • Portability: Code can usually run on multiple types of systems without modification.
  • Rapid Development: Offers faster development time due to its simplicity and available libraries.

Low-Level Languages

What are Low-Level Languages?

Low-level languages include assembly language and machine code. They provide more control over hardware and are specific to a particular type of processor or target hardware architecture.

Advantages

  • Performance: More efficient execution, as they are closer to the hardware.
  • Control: Greater control over system resources and memory.

Limitations

-Machine specific: Low level languages like assembly language is not portable because the data is stored in computer registers, and the computer has to know the different sets of registers.

Code Translators: Compilers and Interpreters

  • Source code refers to code written in a particular programming language.
  • Translators have the responsibility of converting your source code to the machine language. This is also known as binary. Remember ones and zeros. We may refer to the binaries as Object Code , the Program or a common word today: App.

Compilers

  • A compiler is a type of translator that converts high-level programming language code into machine code.
  • Example languages that use compilers: C, C++, Java (to bytecode).

How Does a Compiler Work?

  1. Lexical Analysis: The compiler breaks down the code into small parts, like words and symbols. Think of it like reading a sentence and identifying nouns, verbs, and punctuation.
  2. Syntax Analysis: Now, the compiler checks if these small parts follow the right grammar rules of the programming language, similar to making sure a sentence is grammatically correct.
  3. Semantic Analysis: This step is like understanding the meaning of the sentence. The compiler makes sure that the code makes logical sense.
  4. Intermediate Code Generation: The compiler then rewrites your code into a ‘middle-ground’ language. This is a step between your original code and the final machine code.
  5. Optimization: Here, the compiler makes this ‘middle-ground’ code better and more efficient, similar to editing a draft to make a final, polished version.
  6. Code Generation: The compiler translates the optimized ‘middle-ground’ code into machine code, which is the language that computers understand directly.

Features of Compilers:

  • Converts the entire program in one go.
  • Execution is fast as the translation is done beforehand.
  • Error detection happens before execution.
  • The output is an executable file that can be run independently.

Interpreters

  • An interpreter is a type of translator that converts high-level programming language code into machine code on the fly, during program execution.
  • Example languages that use interpreters: Python, Ruby, JavaScript.

How Does an Interpreter Work?

  1. Reads the Code: Takes the source code one line or statement at a time.
  2. Lexical and Semantic Analysis: Similar to a compiler, it breaks the code into tokens and checks the syntax.
  3. Execution: Immediately executes the translated instructions after the semantic analysis.

Features of Interpreters:

  • Translates and executes the program line by line.
  • Slower execution compared to compiled programs, as translation happens at runtime.
  • Errors are detected during runtime.

Summary

  • Compiler: Translates the entire program at once into machine code, creating an executable file. Suitable for programs where performance is critical.
  • Interpreter: Translates and executes code line by line. Useful for development environments where immediate feedback is needed.

Each type of translator has its own use cases and advantages, and the choice between a compiler and an interpreter often depends on the requirements of the specific programming task at hand.

Von Neumann Architecture

The Von Neumann architecture describes a design model for a digital computer using a processing unit and a single separate storage structure to hold both instructions and data. It is a foundational model for modern computer design.

Key Components

  1. Central Processing Unit (CPU): Acts as the brain of the computer. It performs calculations and logical operations. The CPU is further divided into two main parts:

    • Arithmetic Logic Unit (ALU): Performs arithmetic and logical operations.
    • Control Unit (CU): Directs operations within the processor, coordinating how data moves through the CPU.

  2. Memory: Stores both data and the instructions that the CPU uses. Memory in the Von Neumann architecture is typically volatile, meaning it requires power to maintain its state.

  3. Input/Output (I/O) Mechanisms: These are devices that allow the computer to interact with the external world, such as keyboards, mice, displays, and printers.

  4. Registers: Small, fast storage locations within the CPU. They temporarily hold data that the CPU is currently processing. Some common types of registers include:

    • Accumulator: Stores intermediate arithmetic and logic results.
    • Program Counter (PC): Holds the memory address of the next instruction to be executed.
    • Instruction Register (IR): Holds the instruction currently being executed.
    • Memory Address Register (MAR): Holds the address of the memory location to be accessed.
    • Memory Data Register (MDR): Holds the data to be written to or read from the memory.

  5. Bus: A communication system that transfers data between components inside a computer. It connects the CPU, memory, and I/O devices.

Two’s Complement

Two’s complement is a binary number system that allows for the representation of both positive and negative integers. In this system, the leftmost bit is typically used as the sign bit: ‘0’ indicates a positive number and ‘1’ indicates a negative number. The value of a negative number in two’s complement is obtained by inverting all the bits of its positive counterpart and then adding 1 to the result.

How to Calculate Two’s Complement

  1. Positive Numbers: Represented as their binary equivalent.
  2. Negative Numbers: To find the two’s complement of a negative number:
    • Convert the absolute value of the number to binary.
    • Invert the digits (change 0s to 1s and 1s to 0s).
    • Add 1 to the inverted number.

Why Two’s Complement is Used

  1. Simplifies Arithmetic Operations: Two’s complement allows the use of the same binary addition operation for both addition and subtraction, which simplifies the design of arithmetic circuits in computers.

  2. Efficient Storage Utilization: It provides an efficient way to represent and manipulate signed numbers in binary, maximizing the use of available bits.

  3. Compatibility with Binary Systems: It aligns well with the binary nature of digital systems, making it a natural choice for computer arithmetic.

Creation of Lists in Python

In Python, there are several ways to create lists. Find a review of some of the methods below:

  1. Using Square Brackets []:

    • This is the most common way to create a list in Python.
    • Example: my_list = [1, 2, 3, 4]

  2. Using the * Operator (Repeating Lists):

    • Create a list with repeating elements.
    • Example: repeated_list = [0] * 5 (Results in [0, 0, 0, 0, 0]).

  3. Appending to an Empty List:

    • Start with an empty list and add elements using append().
    • Example:
      my_list = []
for i in range(0, 105, 5):
    my_list.append(i)

  4. Using str.split() Method (For Strings):

    • Split a string into a list based on a delimiter.
    • Example: words = "Hello,World".split(",") (Splits the string into a list with two elements: ['Hello', 'World']).

  5. 2D Lists (List of Lists):

    • Creating lists within a list.
    • Example: nested_list = [[1, 2], [3, 4], [5, 6]]
