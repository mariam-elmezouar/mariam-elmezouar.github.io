Introduction

Pendant ce cours, vous travaillerez dans un environnement virtuel à distance. Les instructions ci-dessous vous guideront à travers la configuration requise au début de chaque exercice en laboratoire. Prenez le temps de comprendre chacun d’eux car cela améliorera votre efficacité pendant les nombreuses heures que vous passerez au laboratoire.

Objectifs

Ce laboratoire vise à:

  • Vous aider à comprendre l’utilisation d’un environnement virtuel à distance
  • Configurer les interfaces réseau
  • Utilisez les commandes Linux de base pour l’administration du système et la configuration du réseau

Obtention du privilège root

De nombreux outils utilisés dans ce cours nécessitent des privilèges root. En général, lorsque vous avez besoin du privilège root, vous avez deux options, soit exécuter un programme en tant que root, soit basculer vers l’utilisateur root. À moins que vous n’exécutiez des fonctions d’administration système, cette deuxième option est dangereuse et inutile car elle vous donne la capacité (et la responsabilité) de causer des dommages.

Par défaut, dans ce cours, vous serez connecté en tant qu’utilisateur «eee330» et la machine virtuelle Linux a été configurée de telle sorte que vous n’avez jamais besoin d’être utilisateur root. Vous recevrez des instructions spécifiques (en utilisant sudo command X) chaque fois que vous aurez besoin d’exécuter une commande qui nécessite des privilèges root.

Considérez pourquoi ce n’est pas une bonne idée de simplement passer à l’utilisateur root:

  • Premièrement, vous avez un accès complet au système de fichiers et pouvez donc facilement (par accident) endommager la configuration du système d’exploitation ainsi que ses applications
  • Deuxièmement, tout programme exécuté pendant que vous êtes root est exécuté avec le privilège root. Il devrait devenir évident pour vous dans ce cours que ce n’est pas une bonne idée du point de vue de la sécurité.

Comprendre l’infrastructure réseau du laboratoire

Bien que nous opérions dans un environnement virtuel à distance, l'infrastructure physique normalement utilisée a été répliquée dans l'environnement virtuel (principalement).

En vous référant à la figure 1 ci-dessus, passons en revue la terminologie qui sera utilisée pendant les exercices et les laboratoires de ce cours.

  • Réseau d’équipe (Team Network).

    Nous désignerons la configuration de réseau établie (par vous) dans l’environnement virtuel VMware comme le «réseau d’équipe».

  • Réseau Commun (Common Network).

    Nous désignerons la configuration réseau partagée par tous les environnements virtuels des étudiants le «réseau commun».

    • Sur le campus, le réseau commun est un concentrateur et un serveur physiques auxquels tous les postes de travail des étudiants peuvent se connecter.
    • Bien que le concentrateur commun et la machine virtuelle du serveur aient été répliqués virtuellement, l'imprimante ne l'a pas été. Ne tenez pas compte de celle-ci.

  • Commutateurs virtuels (Virtual Switches).

    Il existe deux commutateurs virtuels au sein du réseau d’équipe:

    • Commutateur virtuel interne. (Internal Virtual Switch)

      Le «commutateur virtuel interne» sera utilisé pour connecter les instances de VM entre elles au sein du réseau d’équipe.

    • Commutateur virtuel externe. (External Virtual Switch)

      Le “commutateur virtuel externe” sera utilisé pour connecter les instances des VM au réseau commun (via le panneau de brassage).

  • Mappage des adaptateurs réseau aux commutateurs virtuels

    • Chaque VM est préconfigurée avec deux adaptateurs / interfaces réseau.
    • Sur chaque machine virtuelle, nous utiliserons une carte / interface réseau pour se connecter au commutateur virtuel interne et une autre pour se connecter au commutateur virtuel externe.

Configuration IP

Avant chaque exercice ou laboratoire, les adaptateurs / interfaces réseau au sein du réseau d’équipe doivent normalement être configurés. Suivez les étapes ci-dessous pour chaque type de machine virtuelle.

  • Configuration de l’adaptateur réseau Windows VM

    • Ouvrir la VM Windows
    • Selectionnez start -> Control Panel -> Network and Sharing Center -> Change adapter settings
    • Si l’adaptateur nommé External ne dit pas Disabled, cliquez dessus avec le bouton droit de la souris et sélectionnez Disable. Nous ne voulons utiliser qu'un seul adaptateur à la fois.
    • Cliquez avec le bouton droit sur l'adaptateur nommé Internal et sélectionnez Enable pour activer cet adaptateur.
    • Cliquez à nouveau avec le bouton droit sur l'adaptateur et sélectionnez Status.
    • Dans l’onglet General, selectionnez Properties -> Internet Protocol version 4 (TCP/IPv4) -> Properties et selectionnez. Use the following IP address:
      • Configurez votre adresse IP. Rappelez-vous que la variable x est votre numéro de machine hôte qui a été enregistré au début de cet exercice.
        • 10.12.10x.20
      • Configurez le masque de sous-réseau.
        • 255.255.255.0
      • Vous n’avez pas à vous soucier des paramètres de passerelle ou DNS pour cet exercice.
    • Selectionnez Close pour appliquer ces paramètres; soyez patient, cela pourrait prendre du temps.
    • Affichez mais ne définissez pas les paramètres de l’adaptateurExternal:
      • Faites un clic droit sur l’adaptateur et sélectionnez status. Affichez l’onglet General.
        • L’état de la connexion est probablement répertorié comme Connectivité limitée ou inexistante.
        • Ne modifiez pas les paramètres pour le moment.
    • Revérifiez vos résultats:
      • Ouvrez une invite de commande (si vous avez oublié comment, revenez à une étape précédente du préambule).
      • Le programme ipconfig peut être utilisé pour afficher et modifier les paramètres de l’adaptateur réseau.
      • tapez ipconfig /?
        • Révisez les utilisations de ce programme
      • Tapez ipconfig /all
        • Confirmez que les paramètres affichés pour les cartes réseau sont ceux définis dans les étapes précédentes.

  • Configuration de l’adaptateur réseau Linux VM

    • Ouvrez la machine virtuelle Linux
    • Affichez et configurez les paramètres de l’interface internal:
      • Ouvrez un terminal.
      • Le programme ip peut être utilisé pour afficher et modifier les paramètres de l’interface réseau.
      • Exécutez ip avec le commutateur d’aide -h. Revisez les utilisations de ce programme.
      • En utilisant ip, listez toutes les interfaces: ip addr
      • L’une des interfaces répertoriées doit être nommée Internal. Cette interface ne doit pas avoir d’adresse IPv4 attribuée. L’interface doit également être down, ce que vous pouvez dire en raison de l’absence de UP dans le <> à côté du nom de l'adaptateur. Notez si l'interface externe s'affiche actuellement comme étant UP.
        • Si l’interface Internal n’est pas répertoriée, c’est probablement OK. Essayez quand même l'étape suivante.
      • Désactivez l'interface External, activez l'interface Internal et définissez son adresse IP:
        • sudo ip link set External down
        • sudo ip link set Internal up
        • sudo ip addr add 10.12.10x.10/255.255.255.0 dev Internal
      • Confirmez les paramètres de l'interface Internal et affichez mais ne définissez pas l'interface External:
        • Réexécutez ip addr. Observez le nombre total d'interfaces répertoriées.
          • L'une est l'interface loopback, lo
          • L'une est utilisée pour la connexion au commutateur virtuel interne, Internal (décrit ci-dessus) et doit maintenant être configuré selon les instructions ci-dessus.
          • Notez l'interface External. Est-elle attribuée une adresse IP? _ _ (Ne modifiez pas les paramètres pour le moment.)

  • Effectuez un auto-test du commutateur virtuel interne

    • A partir d’un terminal dans la machine virtuelle Linux, ping la machine virtuelle Windows:
      • ping _Windows_VM_IP_ (utilisez l’IP interne de la machine virtuelle Windows appropriée menionné ci-dessus).
      • Y a-t-il une connectivité ? __
      • Annuler ping en tapant Ctrl-c
    • À partir de l’invite de commande dans la machine virtuelle Windows, envoyez une requête ping à la machine virtuelle Linux via le commutateur virtuel Internal:
      • ping Linux_VM_IP (rappelez vous l’IP Linux VM appropriée ci-dessus).
      • Y a-t-il une connectivité ? _ _
      • Il n’est pas nécessaire d’annuler, car le ping de Windows est configuré pour n’envoyer que 4 pings.

Transfert de fichier

Vous pouvez déplacer des fichiers entre vos postes de travail et un ordinateur externe via un serveur FTP sécurisé. Il n'est pas nécessaire de le faire maintenant et cela ne fonctionnera pas avec la configuration actuelle, mais conservez ces instructions pour référence future:

  • Tout d'abord, l'interface External sur la machine virtuelle en question doit être configurée correctement. Vous apprendrez à faire cela dans les futurs laboratoires.
  • Pour accéder à partir d'un ordinateur externe, téléchargez un client SFTP tel que WinSCP et accédez au serveur SFTP en utilisant ce qui suit: sftpuser@ugrad.cns.ece-rmc.ca. Le mot de passe sera donné séparément.
  • Pour accéder à partir de la machine virtuelle Windows, ouvrez une fenêtre de ligne de commande et utilisez psftp pour accéder au même serveur à sftpuser@10.12.0.9. Le mot de passe sera donné séparément.
  • Pour accéder à partir de la machine virtuelle Linux, ouvrez un terminal et saisissez sftp sftpuser@10.12.0.9.
  • Vous pouvez charger et télécharger des fichiers à partir du dossier de upload.
  • Notez qu'il existe une limite de taille pour les téléchargements. Plus important encore, notez que le dossier des téléchargements est vidé toutes les heures. Ne laissez pas les choses importantes ici!

Comment terminer votre session

Les conseils ci-dessus devraient vous aider à maîtriser l’utilisation efficace de l’environnement virtuel du laboratoire Computer Network Security du CMR. Jusqu’à ce que vous vous familiarisiez avec ces instructions, consultez-les au début de chaque période de laboratoire.

Laissez toujours le CNS-VE dans la même configuration que vous l’avez trouvée; rappelez-vous que de nombreux autres étudiants utilisent également l’infrastructure.

Lorsque vous avez terminé chaque laboratoire:

  • Accédez à la machine virtuelle Windows et désactivez l'adaptateur Internal.
    • Dans les paramètres réseau (network settings), cliquez avec le bouton droit sur l'adaptateur nommé Internal et sélectionnez Disable pour désactiver cet adaptateur. Revenez dans le laboratoire si vous ne savez pas où le trouver.
  • Accédez à la machine virtuelle Linux et désactivez l'interface Internal
    • sudo ip link set Internal down

Les machines virtuelles sont représentées sous forme de fichiers dans VMware; une machine virtuelle en cours d’exécution prend beaucoup d’espace mémoire, nous devons donc prendre soin de ne pas laisser les machines virtuelles s’exécuter lorsqu’on en a pas l’utilité. Il est également important de permettre au système d’exploitation invité (Windows ou Linux) de s’arrêter correctement afin qu’il puisse nettoyer correctement sa session. Lorsque vous avez terminé un laboratoire,vous devez:

  • Éteindre la machine virtuelle Windows.
    • Sélectionnez start , puis sélectionnez Shut Down.
  • Éteindre la machine virtuelle Linux.
    • Sélectionnez le gecko -> Log Out -> Shut Down.
  • Fermez vos fenêtres VMRC. Notez que si vous fermez VMRC sans arrêter la machine virtuelle, la machine virtuelle continuera à fonctionner. Comme mentionné, cela consomme les ressources nécessaires aux autres classes. Arrêtez-les lorsqu'ils ne sont pas utilisés.
