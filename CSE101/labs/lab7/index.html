Lab 7: Hangman and Treasure hunt

CSE101 Fall 2023

Introduction

During this lab, you will implement the game hangman and a treasure hunt.

Requirements

Carefully read and follow the instructions. Points will be deducted for working solutions that do not follow the requirements.

Your code needs to follow good coding habits, including proper commenting.

At the top of your file, clearly indicate your name and student number.

Getting Setup

Please follow these setup instructions.

In PyCharm, create a new project, download, and add the following files to it:

Files to Download:
  1. randomword.py: Contains a function `random_word()` that returns a randomly chosen common UK English word. The word list is based on the most common thousand words with a few omissions. All of the words are lower-case only, and none contain any punctuation (e.g., there are no words with apostrophes, like "don't", which otherwise would have made the list. The longest word in the list is 'circumstances' which is 13 letters long. DO NOT MODIFY THIS FILE.
  2. gogographics.py: A slightly modified gogographics library file for the hangman game. USE THIS VERSION OF THE FILE in this lab, instead of the old one.
  3. hangman.py: a starter file where some of the tedious setup has already been done for you. Your implementation of the game will go in this file.
  4. treasure-hunt.py: a starter file for the treasure hunt problem. Your implementation of the problem will go in this file.

Once you have downloaded these files, you can start working on your lab.

Part 1: The Hangman Game

Absolute Musts

Write an on-screen hangman game. A video illustrating the desired product can be found here.
  1. All the setup elements in the starter file hangman.py must be used.
  2. When the player clicks an available (non-greyed-out) letter, the letter is greyed out and one of two things happens:
    • If the letter is found in the word, all the corresponding blanks in the guess area are filled with that letter.
    • If the letter is not found in the word, the next body part of the hangman appears on the hanging tree.
  3. If the player clicks a greyed-out letter, the click is ignored.
  4. If the player clicks anywhere that isn't on a letter, the click is ignored.
  5. If the player guesses all the letters in the word, the game is over; display a message that the player won.
  6. If the player makes too many wrong guesses and all the available body parts are displayed on the hanging tree, the game is over; display a message that the player lost.

Room for Creativity

Change the colors, fonts, sizes, positions, etc. Change the rendering of the hangman. Animate the appearance of the body parts. Have the hangman fall apart or explode off the tree after he has completely appeared. Give the player an opportunity to request a free letter.

Anything else you can think of is fair game, as long as you respect the absolute musts above, and the spirit of the game.
