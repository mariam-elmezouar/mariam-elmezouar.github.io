Labo 7: Tips

GEF330 hiver 2021

Conception

Lisez attentivement les instructions de laboratoire .

Les documents de conception sont destinés à vous guider tout au long du processus de transformation des exigences et des contraintes répertoriées dans les instructions de laboratoire en un réseau opérationnel.

  • Le résumé des communications vous indique quels hôtes doivent être en mesure d'établir des connexions avec d'autres hôtes. Toutes les machines virtuelles, y compris les pare-feu, doivent apparaître sur ce résumé, mais cela concerne le trafic vers ou depuis le pare-feu (comme SSH), et non le trafic passant par le pare-feu.
  • Le diagramme logique est basé sur ITSG-38 et vous donne une structure logique, prenant en compte les exigences de sécurité et les fonctions de chaque machine virtuelle. Ce n'est pas une représentation de la disposition physique de votre réseau, mais il vous indique quelles zones vous aurez et quels hôtes appartiennent à chaque zone.
    • Votre diagramme logique devrait ressembler beaucoup aux exemples de la conférence sur le zonage de sécurité. Consultez les exemples de tables réseau pour voir quels services appartiennent à quelle zone.
    • Rappelez-vous que chaque zone doit avoir au moins un sous-réseau.
  • Maintenant que vous savez quels ordinateurs appartiennent à quelle zone, vous pouvez démarrer votre diagramme physique Commencez par vos zones et déterminez comment vos pare-feu connecteront vos zones entre elles. Vous ne devez pas placer de zone entre les pare-feu, cela rend le routage extrêmement complexe.
    • Une fois que vous avez dessiné vos pare-feu et vos zones, ajoutez des VM individuelles dans chaque zone.
  • En sachant combien de zones vous avez et combien d'hôtes se trouvent dans chaque zone, complétez votre document de conception de sous-réseau. Cela n'a pas besoin d'être un diagramme de votre réseau, juste une liste de vos sous-réseaux avec les détails pertinents (adresse réseau en notation CIDR, masque de sous-réseau, adresse de diffusion, plage IP disponible.)
  • Sur votre diagramme physique, ajoutez des adresses IP en notation CIDR à toutes les interfaces d'hôte et de pare-feu. D'autres détails de sous-réseau tels que les adresses de diffusion et les plages IP complètes ne sont pas requis sur le diagramme physique.
  • Reportez-vous à votre résumé des communications. Pour chaque connexion qui doit être autorisée, commencez par la boîte source sur votre diagramme physique. Accédez à la zone de destination. Chaque fois que vous passez un pare-feu, notez la règle dont vous avez besoin. ('Firewall2, 192.168.0.5-> 192.168.0.25, SSH Port 22') Parfois, vous pouvez avoir besoin d'adresses IP spécifiques, d'autres fois, vous pouvez utiliser des adresses réseau pour accorder l'accès à des sous-réseaux entiers.
  • Une fois que vous avez une liste des règles de pare-feu requises pour chaque pare-feu, transformez-les en commandes appropriées iptables.

Cas de test

Vos cas de test sont extrêmement importants. C'est ainsi que vous confirmez que votre réseau répond aux exigences de conception. Toutes les possibilités de votre résumé des communications doivent être testées, tant pour les connexions autorisées que pour les connexions restreintes.

  • Pingn'est pas le meilleur outil de test ici. Il confirme uniquement si ICMP est autorisé ou bloqué.
  • Hping3est mieux. Vous pouvez tester TCP sur des ports spécifiques.
  • Tester avec une application particulière est encore mieux. Utilisez SSH, des navigateurs Web ou des navigateurs Web avec des proxys activés pour tester des cas d'utilisation particuliers.
  • Vous devez confirmer que chaque connexion autorisée réussit sur le port souhaité.
  • Il n'est pas nécessaire de confirmer que les 65 533 autres ports sont bloqués. Si une connexion doit être bloquée entre deux hôtes, testez un ou deux ports TCP pour confirmer.
  • Documentez vos cas de test. Ils valent beaucoup de points.

Implémentation

Il est recommandé de créer votre réseau dans l'ordre suivant:

  1. Configurez toutes les interfaces requises.
  2. Activez le transfert de paquets sur les machines virtuelles agissant en tant que routeurs.
  3. Configurez les passerelles par défaut et les routes statiques. Toutes les machines doivent avoir des passerelles par défaut, mais seuls vos routeurs doivent avoir des routes statiques. Les routes statiques sur les hôtes sont une mauvaise idée.
  4. Testez minutieusement les connexions entre toutes les machines virtuelles . Même celles qui ne sont pas censés communiquer selon votre résumé des communications. Ne continuez pas tant que toutes les connexions n’ont pas abouti.
  5. Basculez les pare-feu sur Default DROPet configurez vos règles de pare-feu.
  6. Testez les connexions en fonction de vos cas de test.
  7. Dépannez si nécessaire.

Conseils de dépannage

  • Lorsque vous rencontrez des problèmes de connectivité, effectuez le dépannage de la source à la destination et vice-versa.
    • Utilisez tcpdumpsur chaque interface le long du chemin que les paquets sont censés parcourir. Utilisez hping3ou d'autres outils pour envoyer des paquets de test.
    • Si vous voyez des paquets sur une interface d'un routeur / pare-feu mais pas sur l'autre, votre problème réside sur ce routeur / pare-feu.
    • Pour exclure un problème de routage, définissez la stratégie de pare-feu sur ACCEPT. Si le problème disparaît, c'est un problème de pare-feu. Si cela continue, c'est un problème de routage. Si les paquets commencent à se déplacer à travers les deux interfaces de ce routeur / pare-feu mais que vous ne parvenez toujours pas à établir une connexion, vous avez plusieurs problèmes. L'un d'eux était le pare-feu que vous venez de désactiver. L'autre nécessitera un dépannage supplémentaire.
  • Pour tester le proxy Web, ouvrez Mozilla Firefox, accédez à Preferenceset faites défiler vers le bas. Ouvrez les paramètres réseau et définissez l'adresse IP et le port du proxy.
    • Vous pouvez également utiliser hping3.
  • Pour tester l'accès aux serveurs Web externes, utilisez le serveur à l'adresse 10.12.0.1.
  • Pour tester l'accès à votre serveur Web à partir de réseaux externes, utilisez votre poste de travail Admin via son interface externe.

Rappels
