Use of comments: Starting this lab, you will be required to use comments in your code. What are comments? comments are a programmer-readable explanation or annotation in the source code of a computer program that will be generally ignored by the Python compiler. They are used to explain the code, and make it more readable (read more here).

There are two types of comments:

  • Comments that start with a #: use them to explain steps in your algorithms. Example:
    • # Assign a value to a variable
age = 30  # This variable stores the age of a person

# Perform a calculation
double_age = age * 2  # Multiply the age by 2

# Display the result
print(f"The person's age is {age}.")  # Print the age
print(f"Twice the age is {double_age}.")  # Print double the age

    Use comments when necessary for clarity. Do not overuse.

  • Multiline comments with triple quotes: use them at the begining of your Python file as follows: 
    •         """
        Name: firstname lastname        
        Student number: xxxxxxx
        """

1.   Syracuse sequence

The Syracuse sequence is generated by starting with a natural number and repeatedly applying the following function until reaching 1:

syr(x) = x/2 if x is even, 3x + 1 if x is odd

Write a Python script called syracuse.py to compute and display the Syracuse sequence for a value provided by the user. Your program must display its output exactly as illustrated below (including all punctuation and spacing) when given an input of 3. 

Give me a starting number: 3
The Syracuse sequence for 3 is 
3, 10, 5, 16, 8, 4, 2, 1

Of course, it must display similarly-formatted and correct output for any other natural number given by the user!

Hint: You can append the numbers of the sequence in a list as you compute them (learn more about Python lists here, here, and here). Once done, you can print the list at the end as shown above (read here to explore different ways to print lists.).

2.   Prime test

A prime number is a natural number that has exactly two distinct natural number divisors: 1 and itself. The number 1 is not prime, but 2 is.

A positive whole number n > 2 is prime if no number between 2 and the square root of n (inclusive) evenly divides n. Write a Python script called primetest.py that accepts a value of n as input and determines whether the value is prime using a while loop. If n is not prime, your program should stop searching as soon as it finds the first divisor of n. Here is a sample of the output your program should produce for a number that is prime: 

Give me a number: 67
67 is prime.

and a sample for a number that is not prime: 

Give me a number: 121
121 is divisible by 11.

Your program must correctly handle the cases where the user provides an n of 1 or 2.
