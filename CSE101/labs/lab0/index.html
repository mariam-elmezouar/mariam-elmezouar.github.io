Task 0: Understanding the File System Structure in Windows

Introduction

The file system in Windows is like the digital filing cabinet for your computer. It's how your computer organizes and stores files and folders, making it easier for you to find and manage your information. In this part of the lab, we'll break down the key concepts of the Windows file system structure.

Key Concepts

  1. Drives: Think of drives as the different storage spaces in your computer. They can be physical drives like your main hard disk or external USB drives. In Windows, each drive is typically assigned a letter, like "C:", "U:", and so on.
  2. Folders (Directories): Folders are like digital containers for files. They help you group and organize related files. Just like real folders in a cabinet, digital folders hold files and other folders. Your documents, pictures, and videos are often organized into separate folders for better management.
  3. Files: Files are the actual documents, pictures, videos, and programs stored on your computer. They can be anything from a Word document to a song or a photo. Each file has a name, and it's stored inside a folder.
  4. Path: A path is like a set of directions to find a file or folder. It's made up of the drive letter, folder names, and the file name. For example, C:\Users\YourUsername\Documents\my-file.docx is a path to a Word document on the "C:" drive.

Exploring the File System

Let's use an analogy to understand the file system better:

Imagine your computer is a big city with different neighborhoods (drives). Each neighborhood has houses (folders), and inside each house, you can find things like books, photos, and toys (files).

  1. Navigate Between Drives and Folders:
    • Open "File Explorer" (you can find it in the taskbar or press Win + E).
    • On the left, click on This PC you'll see the drives (C:, U:, etc.). Clicking a drive shows its folders.
    • Click on a folder to see its contents.
  2. Create a New Folder:
    • Locate and open the 'U:' drive. This drive will also be named after your username (refer to the picture below). This drive is unique because whatever you place in it: 1) will still be accessible when you log in later, and 2) its contents can be accessed similarly when logging into a different computer on the RMC network. The 'C:' drive should also keep your files intact, but only if you access them from the same computer each time.
    • Once inside the U: drive, right-click, choose "New," and select "Folder." Name it 'CSE101-labs'. Remember this folder, you can use it to organize all the files related to the course labs
    • Repeat the same process to create a folder called 'Lab0' under 'CSE101-labs'.
  3. Create a Text File:
    • Once inside the folder 'Lab0', right-click, choose "New," and select "Text Document." Name it something like "task0.txt."
  4. Move/copy Files and Folders:
    • Right-click a file/folder, select "Cut" (to move) or "Copy" (to create a copy), go to another folder, and right-click to "Paste".
    • Practice dragging a file or folder and dropping it into another folder to move it.
  5. Understand Paths:
    • Look at the address bar when you're inside a folder. It shows the path.
    • Experiment by copying a path and pasting it into the address bar to navigate directly.

Takeaway

Understanding the file system of your computer is crucial for effective organization and management of your files for this lab and for your courses in general. Remember the key concepts: drives, folders, files, and paths. By using File Explorer and practicing basic tasks, you'll become more comfortable with the structure and navigation of your computer's file system.

Task 1: PyCharm - a Python IDE

What is PyCharm?

PyCharm is an Integrated Development Environment (IDE) designed specifically for Python programming. An IDE is like a digital workshop where you write, test, and manage your code more efficiently. PyCharm provides tools and features that help programmers, especially beginners, create Python programs with ease.

Task 2: The Guessing Game with Python

Create a new Python file called playerguess.py and type the program below into it exactly as written.

Do not copy and paste! Part of the point of this lab is to get you used to writing code precisely the way Python needs it entered. PyCharm will give you hints about typos or other errors (e.g., using the wrong indentation, etc.) by red underlining or colouring.


import random

target = random.randint(1, 100)
guess = -1

print("\nI've picked a number between 1 and 100.")
print("Try to guess what it is.\n")

while guess != target:
    guess = int(input("What is your guess? "))
    if target > guess:
        print("higher...")
    elif target < guess:
        print("lower...")

print("\n******\nYou guessed it!\n******")

Once you have the program entered, try to run it. If PyCharm gives you an error message, try to figure out where you've made your mistake. Fix any errors until the program seems to work correctly.
