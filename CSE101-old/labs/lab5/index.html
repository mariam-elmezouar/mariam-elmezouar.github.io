Lab 5: Sorting

CSE101 Fall 2022

Introduction

During this lab, you will implement two sorting algorithms: selection and insertion sorts.

Objectives

The objective of this lab is:

  • To convert a concept (the sorting process) into a complete algorithm.
  • To use MATLAB to solve more complex problems, involving nested loops.
  • To practice programming within the Matlab IDE
  • To test your code

Getting setup

The following starting code contains three files: two function Matlab files (insertionsort.m and selection.m) and one test file (test.m). Keep the three files together in the same folder. Open the three files in Matlab, and try running test.m. Nothing should happen at this point since the sorting functions are not implemented yet. The purpose of the test files is to test the two functions you will be implementing in the rest of the lab.

Few rules to note about this lab:

  • You can work in teams of up to three students. All members should contribute equally. If not, the members will not get the same grade on this lab.
  • At the top of each function file, add a comment as follows %Team members: X, Y, and Z.
  • Part of your grade on this lab will be based on the documentation of your code, i.e. adding comments that explain your logic and what your code is doing. Add comments as needed and as clearly as possible.

Part 1: Selection Sort

Selection sort involves traversing the list to find the smallest number and then swapping it with the element in the first position. Then, we find the second smallest number starting at the second position, and we swap it with the element at the second position. We continue in this manner until the entire list is sorted.

In the function file called selectionsort.m, write the implementation of selection sort.

Use the file test.m to test your code as you go. You may modify test.m to add more tests as you wish.

Part 2: Insertion Sort

Selection sort was not explained in class. Part of your task here is to research and understand the steps of insertion sort. A good place to start is here (there is a little simulation on the web page that you can play with to understand the algorithm).

In the function file called insertionsort.m, write the implementation of insertion sort.

Use the file test.m to test your code as you go. You may modify test.m to add more tests as you wish.
